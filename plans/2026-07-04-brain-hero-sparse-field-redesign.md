# Brain-hero: ре-дизайн под «разрежённое глубокое поле спектра» (v2 — усиленный)

Дата: 2026-07-04. Основа: дизайнер-критика (`reference/clone-brain.png` vs `reference/live-00-hero.png`), `AUDIT.md`, `REFERENCE-GAP.md`.
Цель: не «догнать» референс, а **перегнать** — та же ДНК (разрежённость/спектр/глубина), плюс кинематографичность, которой у Dala нет.

---

## Диагноз (одной строкой)

Сейчас = буквальный плотный **золотой** мозг-блоб. Живой Dala = разрежённое **полноспектровое** поле с большой глубиной, мозг — эмерджентное сгущение на скролле, а не картинка на 0%. Позиции верные, презентация противоположна.

## Ключевая структурная идея v2 — ДВА СЛОЯ

Главный апгрейд. Сейчас один набор частиц морфит brain→bulb→earth. У референса на самом деле **два независимых слоя**, отсюда его «живость»:

- **LAYER A — SHAPE** (~7000): морф brain→scatter→bulb→earth. Это «смысл».
- **LAYER B — AMBIENT** (~4000): НИКОГДА не морфит. Всегда дрейфует по всему вьюпорту, спектр, широкий Z. Экран **никогда не пустой**, всегда есть передние крупные треугольники и глубина. Это «атмосфера».

Один слой = либо пусто между секциями, либо блоб. Два слоя = постоянное живое поле + читаемая форма поверх. Это то, что делает Dala дорогим, и то, чего у нас нет.

## Точные параметры (готовые к вставке значения)

Всё в `index.html`, если не указан `build-shapes.js`.

**Палитра (строки ~199-210)** — заменить `pickHex(goldBias)` на equal-mix, 6 цветов:
```
GOLD 0xffbe3c/0xffd066 ·20%  PURPLE 0x7b61ff/0x8a5cfc ·20%  TEAL 0x23b99a/0x2fc79e ·18%
BLUE 0x4f9bff/0x5aa9ff ·16%  MAGENTA 0xe0559a/0xff6fae ·14%  WHITE 0xffffff/0xf3eee6 ·12%
```
Убрать аргумент `goldBias` целиком (единственный вызов — строка ~252).

**Цвет-boost / размер (строки ~250-257):**
- `boost = 0.85 + ridge*0.9` (сейчас `0.62 + ridge*1.35` — грязнит; новый не моет цвет).
- `pickHex()` без bias.
- Размер: `(0.7 + pow(rand, 3.2) * 7.5) * (0.7 + ridge*0.6)` — степень 3.2 = много мелких + редкие огромные передние.

**Scatter-разброс (строка ~247):** `640 / 400 / 300` (сейчас `540/340/230`) — шире XY (полный вьюпорт) + глубже Z (парал­лакс).

**Bloom (строка ~233):** `UnrealBloomPass(..., 0.55, 0.6, 0.16)` (сейчас `1.08,0.5,0.07`). strength вниз, threshold вверх → горят только ядра, край хрустит.

**Camera-Z пролёт (animate-loop, ~строка 369):** добавить `camera.position.z = 600 - scrollP*230;` (сейчас статична 560) — камера летит сквозь поле, передние треугольники проносятся.

**Hero-хореография (vertex shader timeline, ~строки 282-290):**
```
p<0.05:  wB=0.60 wSc=0.40            // airy field, много чёрного
0.05-0.10: condense wB 0.60→1.0      // reveal-сборка
0.10-0.14: wB=1.0                    // мозг собран (payoff)
0.14-0.26: brain→explode             // (как было; всё от 0.26 не трогаем — панели синхронны)
```

**Ambient-слой (новый, v2):** второй `THREE.Points` ИЛИ ветка в том же geo с флагом `aAmbient=1`:
- позиция = только `aScatter` (не участвует в morph-миксе весов), всегда дрейф `amp` больше.
- ~4000 точек, спектр тот же, Z-разброс ±360, размер тот же степенной.
- рендерится всегда, поверх/под shape — additive, так что просто складывается.

## Фаза 7 (2026-07-04) — объёмный мозг (запрос: «глубже/объёмнее»)

Проблема: мозг читался плоским. Причина — Z в `build-shapes.js` = **random cloud** внутри bulge, гравюра-гири не в 3D.

Решение (build-shapes.js): Z = округлая эллипсоид-оболочка `round*0.28` + **гири-heightmap** (`relief = max(0,b-0.5)*0.30`, гребни выпирают, борозды тонут) + оболочка front/back 55/45 (see-through wireframe-объём) + тонкая толщина. Вращение (index.html) subtle 0.22 — объём от рельефа, не от кручения.

Итерации (все прогнаны через verify-shot + histogram):
- **vol1** глубина 0.46 + кручение 0.42 → яркий хаос-шар, силуэт потерян, near-black 47.9% → откат.
- **vol2** front-heavy 0.72 → пересвет по центру (white 24.5%), борозды утоплены → откат.
- **vol3 (принято)** глубина 0.28, оболочка 0.55, bloom 0.55→0.42/0.20, near-cap 2.05→1.4.

**vol3 vs предыдущий (p5) vs baseline vs живой Dala:**

| Метрика | baseline | p5 (плоский) | **vol3 (объём)** | живой Dala |
|---|---|---|---|---|
| gold | 85.3% | 19.7% | **21.2%** | 55.6% |
| near-black | 56.9% | 64.3% | **62.3%** | 67.6% |
| white (пересвет-маркер) | 5.0% | 16.8% | **15.9%** | 14.5% |
| console errors | — | 0 | **0** | — |

Объём добавлен, палитра/чистота held (white 15.9% ≈ p5, не blowout). near-black 62.3% (−2 от p5: рельеф+оболочка заполняют чуть плотнее; в полосе референса ~67.6, приемлемо).

Также: `verify-shot.js` переведён на `waitUntil:"domcontentloaded"` — устраняет зависание при медленном three-CDN.

**Итоговая приёмка 7 дельт:** цвет ✓, спрайт ✓, глубина ✓, ambient ✓, композиция ✓, текст-секции ✓, объём ✓; near-black 62.3% (target 70 — мягко не добит, в полосе живого 67.6, осознанно из-за ambient+рельефа).

## Фаза 8 (2026-07-04) — Земля-глобус (запрос: «сфера как земля, США спереди»)

Было: earth-сфера = радужный фаззи-шар, не читается как Земля.

Решение:
- **build-shapes.js**: 18 metaball-континентов (Сев/Юж Америка, Африка, Европа, Азия, Австралия, Антарктида), суша 82%/океан 6%; `FRONT=190°` разворот → США смотрит в камеру (+z); экспорт `land[]` (1=суша).
- **index.html шейдер**: атрибуты `aLand`/`aAmb`; земной тинт `mix(sea#0a2fa3, land#61b74d)` gated на `wE*(1-aAmb)`; задняя полусфера гаснет `front=smoothstep(-90,30,aEarth.z)`; треугольники земли ×0.6; **ambient почти убран** `*= 1-wE*aAmb*0.92` (иначе радуга-ambient забивала землю — 2 итерации: earth1 rainbow-ball, earth2 dilute, earth3 принят).

Регрессии нет: всё gated на `wE` (earth-вес), в hero/brain/bulb `wE=0`. Проверка hero после правок: gold 18.5%, near-black 62.1% = vol3 в пределах шума, 0 console errors. Кадр `reference/earth3-86.png`.
